# 响应式适配指南

## 📱 断点系统

项目使用三级响应式断点系统：

```less
// 桌面端（默认）: ≥ 1025px
// 平板端: 769px - 1024px  
// 移动端: ≤ 768px

@mobile: ~'(max-width: 768px)';
@tablet: ~'(min-width: 769px) and (max-width: 1024px)';
@desktop: ~'(min-width: 1025px)';
```

## 🎯 设计原则

### 1. **桌面优先 → 渐进缩减**
- 默认样式针对桌面端设计（最大、最舒适）
- 平板端适度缩减
- 移动端最紧凑

### 2. **三级适配策略**
```less
.element {
  // 桌面端（默认）
  font-size: 18px;
  padding: 32px;
  
  @media @tablet {
    // 平板端（中等）
    font-size: 16px;
    padding: 28px;
  }
  
  @media @mobile {
    // 移动端（紧凑）
    font-size: 14px;
    padding: 24px;
  }
}
```

## 🎨 交互效果适配

### 悬停效果
```less
.interactive-element {
  // 桌面端：明显的悬停效果
  &:hover {
    transform: scale(1.05);
    filter: brightness(1.1);
  }
  
  @media @tablet {
    // 平板端：较轻微的效果
    &:hover {
      transform: scale(1.03);
    }
  }
  
  @media @mobile {
    // 移动端：禁用悬停，使用按压效果
    &:hover {
      transform: none;
      filter: none;
    }
    
    &:active {
      transform: scale(0.95);
    }
  }
}
```

## 🔧 实际应用示例

### LoadingScreen 组件
- **桌面端**：更大的间距和字体，提供舒适的视觉体验
- **平板端**：适中的尺寸，平衡美观和实用性
- **移动端**：紧凑布局，最大化内容显示区域

### MusicControl 组件
- **桌面端**：16px 内边距，24px 图标，悬停缩放效果
- **平板端**：14px 内边距，22px 图标，轻微缩放
- **移动端**：10px 内边距，20px 图标，按压反馈

### QuoteCard 组件
- **桌面端**：600px 最大宽度，48px 内边距，深阴影
- **平板端**：520px 最大宽度，40px 内边距，中等阴影
- **移动端**：500px 最大宽度，30px 内边距，浅阴影

## 📋 最佳实践

1. **渐进式设计**：从桌面端开始，逐步适配到移动端
2. **一致性原则**：相同类型的元素使用相同的缩放比例
3. **触摸友好**：移动端确保最小 44px 的触摸目标
4. **性能优化**：移动端减少复杂的视觉效果
5. **可读性优先**：确保所有断点下的文字都清晰可读
